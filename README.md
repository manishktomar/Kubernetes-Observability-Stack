# Kubernetes Observability Stack

## Overview

This repository documents a complete open-source observability stack for Kubernetes and cloud-native applications.

The solution provides end-to-end visibility into:

- 📊 Metrics
- 📝 Logs
- 🔍 Distributed Traces
- 🚨 Alerting
- 📈 Dashboards
- 🌐 Service Mesh Monitoring

The stack is built entirely using free and open-source technologies and can be deployed on Kubernetes clusters running on-premises or in the cloud.

# Components

## OpenTelemetry SDK

- Instruments applications
- Generates Metrics
- Generates Logs
- Generates Traces

Think of it as **CCTV cameras installed inside every application**.

---

## OpenTelemetry Collector

Acts as the central telemetry pipeline.

Responsibilities:

- Receive telemetry
- Process data
- Filter data
- Transform data
- Export data to multiple backends

Think of it as a **central courier hub sorting parcels**.

---

## Prometheus

Prometheus collects and stores time-series metrics by scraping configured targets.

Features:

- Pull-based architecture
- PromQL support
- Alert rule evaluation
- Kubernetes native

Used for:

- CPU
- Memory
- Disk
- Network
- Application metrics

---

## Thanos

Extends Prometheus capabilities by providing:

- Long-term storage
- Global querying
- High availability
- Object storage integration

Recommended for production environments with long metric retention requirements.

---

## Fluent Bit

A lightweight log collector for Kubernetes.

Responsibilities:

- Collect container logs
- Parse logs
- Enrich logs
- Forward logs to Loki

Think of it as a **postman collecting letters from every house**.

---

## Loki

Log aggregation system designed specifically for Kubernetes.

Advantages:

- Lightweight indexing
- Cost efficient
- Native Grafana integration
- LogQL support

Stores application and infrastructure logs.

---

## Tempo

Distributed tracing backend from Grafana Labs.

Stores:

- Request traces
- Span relationships
- Service dependencies

Advantages:

- Object storage backend
- No expensive indexing
- Scales horizontally

---

## Grafana

Unified visualization platform.

Supports:

- Metrics
- Logs
- Traces

Provides:

- Dashboards
- Correlation
- Alerting
- Reporting

Think of it as an **airport control tower dashboard**.

---

## Alertmanager

Routes alerts generated by Prometheus.

Supports notifications via:

- Slack
- Microsoft Teams
- Email
- PagerDuty
- Webhooks

Handles:

- Grouping
- Silencing
- Deduplication
- Routing

---

## Node Exporter

Exports Linux host metrics.

Provides:

- CPU usage
- Memory usage
- Disk usage
- Network statistics
- Filesystem metrics

---

## kube-state-metrics

Exports Kubernetes object states.

Examples:

- Deployments
- Pods
- Nodes
- StatefulSets
- DaemonSets
- Jobs
- CronJobs

---

## cAdvisor

Collects container runtime metrics.

Provides:

- CPU usage
- Memory usage
- Filesystem usage
- Network usage

---

## Kiali

Visualization platform for Istio Service Mesh.

Features:

- Service topology
- Request flow
- Traffic graph
- Health status
- Distributed tracing integration

# SLI, SLO and SLA

| Term | Meaning |
|----------|-------------------------------------------|
| **SLI** | Actual measured service performance |
| **SLO** | Target performance objective |
| **SLA** | Contractual commitment to customers |

### Example

- **SLI:** API availability is 99.93%
- **SLO:** Target availability is 99.90%
- **SLA:** Contract guarantees 99.90% uptime
